一

SQL 視觀表 CustSummary 與可更新性

配分 30 分

題目

假設某家銀行定義了一個借貸表格（BORROWER）管理顧客借款的資料，包含借款顧客（ID）、借款編號（LoanNO）和金額（Amount）等三個屬性。請利用 SQL 語法，首先在資料庫系統中定義一個叫做 CustSummary 的視觀表（View），表達每一個顧客的借款總數量和借款總金額。其次，利用此視觀表和 SQL 查詢句，輸出身分證字號為「A123456789」的借款總金額。最後，請說明為何此視觀表只適用於查詢資料，而不適用於建立新的資料。

解題重點

題目要求用 BORROWER(ID, LoanNO, Amount) 建立每位顧客的借款彙總 View。彙總欄位包含借款總數量與借款總金額，因此需要 GROUP BY ID，並使用 COUNT 與 SUM。

SQL 寫法

CREATE VIEW CustSummary AS
SELECT
  ID,
  COUNT(LoanNO) AS LoanCount,
  SUM(Amount) AS TotalAmount
FROM BORROWER
GROUP BY ID;

SELECT TotalAmount
FROM CustSummary
WHERE ID = 'A123456789';

說明

CustSummary 是依顧客身分證字號分組後產生的彙總結果。每個 ID 只會出現一列，LoanCount 表示該顧客借款筆數，TotalAmount 表示該顧客所有借款金額加總。

這個 View 只適合查詢，不適合新增資料，原因是它不是直接對應 BORROWER 的單一基底資料列，而是由多列資料經過 GROUP BY、COUNT、SUM 聚合而成。若要在 View 中新增一列，例如 A123456789, 3, 900000，資料庫無法判斷應該在 BORROWER 中新增幾筆 LoanNO、每筆金額各是多少，因此通常不可更新或不可插入。

二

BCNF 判斷、違反原因與可能問題

配分 30 分

題目

BCNF（Boyce-Codd Normal Form）是設計關聯式資料庫綱要時常遵守的形式，請說明如何判斷一個表格是否符合 BCNF 的規定？若不符合會帶來何種問題？下列定義表格 R 的綱要、函數相依性和三筆範例資料： 

R = (A, B, C, D)
F = {AB → CD, C → B}

A   B   C   D
a1  b1  c1  d1
a1  b2  c1  d2
a2  b1  c1  d3

請根據上述資訊詳加解釋為何該表格不符合 BCNF？並以範例資料明確指出此表格可能產生的問題。

BCNF 判斷原則

一個關聯表格若要符合 BCNF，對於每一個非平凡函數相依 X → Y，左側決定因素 X 都必須是該表格的超鍵。也就是說，任何能決定其他屬性的欄位組合，本身都要能唯一決定整筆資料。

套用到題目 R(A, B, C, D)

題目給定函數相依性：

  • AB → CD：AB 可決定 C 與 D，因此 AB 可決定 A、B、C、D，AB 是候選鍵。
  • C → B：C 可決定 B，但 C 不能決定 A 與 D，因此 C 不是超鍵。

因為 C → B 的左側 C 不是超鍵，卻可以決定非鍵屬性 B，所以 R 不符合 BCNF。

用範例資料指出問題

A B C D
a1 b1 c1 d1
a1 b2 c1 d2
a2 b1 c1 d3

若規則 C → B 成立，則同一個 C 值應只能對應同一個 B 值。但範例中 c1 同時出現 b1 與 b2，表示資料已經可能產生不一致。這正是 BCNF 違反後常見的異常。

可能問題包括：第一，更新異常，若 c1 對應的 B 要改，必須更新多列，漏改就不一致；第二，插入異常，若只想新增 C → B 的規則，卻還沒有 A 與 D，可能無法自然存入；第三，刪除異常，刪除某筆資料時可能把某個 C 對應 B 的事實一併刪掉。

三

資料倉儲 ETL 與同步/非同步資料收集

配分 20 分

題目

資料倉儲（Data Warehouse）架構常被用來整合多種資料來源以進行資料分析，而建立資料倉儲的其中一個重要步驟是進行 ETL（Extract, Transform, Load）。首先，請詳述該步驟的功用。其次，收集資料的時機有同步（Synchronous）和非同步（asynchronous）兩種，請詳述此兩種做法的差別和優缺點。

ETL 的功用

ETL 是資料倉儲建置的重要流程，包含 Extract、Transform、Load。Extract 負責從交易系統、檔案、API、日誌或外部資料源擷取資料； Transform 負責清理錯誤、統一格式、轉換編碼、去除重複、補足缺值、整合主鍵、建立衍生欄位與商業規則；Load 則將處理後資料載入資料倉儲、資料超市或分析模型。

ETL 的核心價值是把分散且格式不一的資料整理成可供查詢、報表、OLAP 與決策分析使用的一致資料。

同步與非同步資料收集

同步 Synchronous

資料來源產生變動時，立即同步到資料倉儲或中介系統。優點是資料新鮮度高、一致性較強，適合即時監控或風險控管。缺點是系統耦合度高，來源系統可能因同步程序而增加延遲，若目標系統異常也可能影響原交易流程。

非同步 Asynchronous

透過排程批次、佇列、事件串流或日誌擷取，在稍後時間處理資料。優點是對來源系統影響較小、可批量處理、容錯與重試較容易。缺點是資料有時間落差，分析結果可能是最終一致而非即時一致，也需要處理延遲、重複與補償問題。

資料字典 Data Dictionary

配分 20 分

題目

資料庫管理系統中有一種特別的資料結構稱作資料字典（Data Dictionary）。請說明資料字典中存放何種資料，並提出至少三類，以及關聯式資料庫系統通常如何實作資料字典。

資料字典存放的資料

資料字典存放的是「描述資料的資料」，也就是資料庫的 metadata，而不是一般業務資料。它讓 DBMS 知道資料表、欄位、限制、索引、權限與物件相依性如何被定義。

  • 綱要與結構資訊：資料表、欄位名稱、資料型態、長度、預設值、View 定義。
  • 限制與關聯資訊：主鍵、外鍵、唯一限制、檢查限制、資料表之間的相依關係。
  • 安全與權限資訊：使用者、角色、授權、物件存取權限。
  • 效能與儲存資訊：索引、統計資料、表空間、資料檔案、分割區資訊。
  • 程式物件資訊：Stored Procedure、Function、Trigger、Package 或排程工作。

關聯式資料庫如何實作

關聯式資料庫通常以系統目錄表或系統 View 實作資料字典。這些資料本身也存放在資料庫中，由 DBMS 自動維護，例如建立資料表時，系統會把表格名稱、欄位型態、限制與索引寫入系統目錄。

使用者通常不直接修改資料字典，而是透過 SQL DDL，例如 CREATE TABLE、ALTER TABLE、CREATE INDEX 讓 DBMS 更新。查詢時則可透過標準的 INFORMATION_SCHEMA 或各資料庫提供的系統 catalog view 查看。

一

DHCP 自動設定與 169.254 位址

配分 25 分

題目

小華買了一台新的筆記型電腦，插上房間的網路線準備參加線上課程。這台電腦必須透過家中的路由器取得正確的網路參數才能連上網際網路。請說明新電腦通常透過路由器內建的何種服務自動取得 IP 位址，以及若 IP 位址為 169.254.1.5 且無法開啟網頁，代表發生什麼問題。

自動取得 IP 的服務

新電腦接上有線網路時，通常會透過路由器內建的 DHCP（Dynamic Host Configuration Protocol，動態主機設定協定）自動取得網路參數。

DHCP 的目的，是自動分配 IP 位址，並提供子網路遮罩、預設閘道與 DNS 伺服器等設定，讓電腦能加入區域網路並連上網際網路，也可避免使用者手動設定錯誤。

169.254.1.5 的意義

若電腦取得 169.254.1.5，代表它沒有成功從路由器的 DHCP 伺服器取得 IP 位址。169.254.x.x 屬於 APIPA（Automatic Private IP Addressing，自動私有 IP 位址），是 DHCP 請求失敗後由作業系統自動產生的位址。

可能原因包括路由器 DHCP 功能未啟用、網路線或網路孔故障、路由器異常，或電腦與路由器之間無法正常通訊。因為沒有取得正確的預設閘道與 DNS，電腦通常無法開啟網頁。

二

蜂巢式行動通訊系統元件與位置更新

配分 25 分

題目

在蜂巢式系統中，行動通訊系統包含 MSC、AUC、EIR、HLR 與 VLR 五個元件。請分別說明其作用，並說明手機移動至註冊區域外時，MS 與當地 BS 之間如何維持通訊功能。

五個元件的作用

  • MSC，Mobile Switching Center：負責通話交換、路由選擇、連線控制，並與其他電話網路或行動網路互通。
  • AUC，Authentication Center：負責使用者身分驗證，確認 SIM 卡與用戶是否合法，防止冒用或非法接取。
  • EIR，Equipment Identity Register：儲存手機設備 IMEI 資訊，用來判斷設備是否合法，例如封鎖失竊或非法手機。
  • HLR，Home Location Register：儲存用戶永久資料，例如電話號碼、服務項目、SIM 身分資料與目前所在的大致位置。
  • VLR，Visitor Location Register：暫存進入該區域的外來用戶資料，讓使用者在非本籍區域也能撥打、接聽與使用服務。

移動到其他城市時的步驟

當手機偵測到目前基地台屬於新的註冊區域，會透過當地基地台向網路發出位置更新或註冊請求。當地 MSC/VLR 接收後，向原本所屬的 HLR 查詢用戶資料，HLR 再將必要資料傳送給新的 VLR。

接著 AUC 進行 SIM 與用戶身分驗證，EIR 可檢查手機 IMEI 是否為合法設備。驗證成功後，新 VLR 暫存小明的用戶與位置資料，HLR 更新小明目前所在的 VLR 位址，舊 VLR 的暫存資料可被刪除。完成後，小明即可在新城市正常使用通訊服務。

三

數位簽章驗證流程與安全特性

配分 25 分

題目

數位簽章運用非對稱加密技術確認電子文件身分與內容。小明先將原始文件做 Hash 得到訊息摘要，再用自己的私鑰加密摘要生成數位簽章，並將原始文件與數位簽章一併寄出。請說明小美收到後的驗證程序，以及此程序可保證哪些資訊安全特性。

小美的驗證程序

小美收到原始文件與數位簽章後，先對收到的原始文件進行相同 Hash 運算，產生新的訊息摘要。接著取得小明的公開金鑰，並用該公開金鑰解開數位簽章，得到小明簽署時加密的訊息摘要。

最後，小美比較兩個摘要：一個是自己重新計算得到的摘要，另一個是從數位簽章解出的摘要。若兩者相同，表示文件未被竄改，且簽章確實可由小明公開金鑰驗證；若不同，則代表文件可能遭竄改或簽章來源不正確。

保證的安全特性

  • 完整性，Integrity：文件只要被修改，Hash 值就會不同。摘要一致表示傳輸後內容未被竄改。
  • 身分鑑別，Authentication：簽章由小明私鑰產生，若能以小明公開金鑰驗證，表示簽章可對應到小明的身分。
  • 不可否認性，Non-repudiation：因為只有小明應持有私鑰，簽章可作為小明曾簽署該文件的證明，使其較難事後否認。

需要注意的是，數位簽章本身主要不保證機密性，因為原始文件通常仍以明文一併傳送；若要保密，還需另行加密文件內容。

四

TCP 三向交握與逾時重傳

配分 25 分

題目

小明使用有線網路觀看高畫質串流影片，播放軟體採用 TCP 確保資料傳輸可靠。請描述影片資料開始傳送前的三向交握流程與目的；若傳送途中部分資料遺失且傳送端未在一定時間內收到 ACK，請說明傳送端會採取什麼動作。

三向交握流程與目的

TCP 建立連線時，第一步由小明的電腦送出 SYN 給串流伺服器，表示想建立連線並告知自己的初始序號。第二步伺服器回傳 SYN + ACK，表示同意建立連線、確認收到用戶端的 SYN，並提供伺服器自己的初始序號。第三步用戶端再回傳 ACK，確認收到伺服器的 SYN。

三向交握的目的，是確認雙方都存在且能互相通訊，同步雙方序號，建立可靠的 TCP 連線，並為後續資料傳輸、確認應答與重傳機制做準備。

資料遺失時的處理

當影片傳送途中因干擾造成部分資料遺失，傳送端在一定時間內沒有收到接收端的 ACK，會進行 逾時重傳（Timeout Retransmission），重新傳送尚未被確認收到的資料封包。

這樣做的目的，是補回遺失資料，確保接收端能收到完整內容，維持 TCP 的可靠傳輸特性，避免資料因封包遺失而出現缺漏或錯誤。

一

時間複雜度漸近符號

配分 20 分

題目

請以時間複雜度之漸近符號觀念回答。前兩小題請以 Θ 與 n 表示時間複雜度；另比較甲演算法 O(n log n) 與乙演算法 O(n^2) 在實際執行時間上何時不一定由甲較快。 

sum=0;
for (i=0; i<n; i=i+2)
  for (j=0; j<i; j++)
    sum=sum+1;
sum=0; i=0;
while (i<n) {
  for (j=i; j<=n; j++)
    sum=sum+1;
  i=i+1;
};

程式片段一

外層迴圈的 i 依序為 0、2、4，一直到小於 n。內層執行次數為 i 次，因此總次數約為 0 + 2 + 4 + ... + (n 附近的偶數)，這是等差級數，總和與 n^2 同階。

所以時間複雜度為 Θ(n^2)。

程式片段二

i 從 0 增加到 n - 1。當 i = 0 時，內層約執行 n + 1 次；當 i = 1 時，約執行 n 次；依此類推，總次數約為 (n + 1) + n + ... + 2。

此總和仍為二次級數，因此時間複雜度為 Θ(n^2)。

O(n log n) 不一定實際較快的情況

可列舉「資料量很小」的測試資料，例如只測試數十筆或數百筆資料。漸近複雜度描述的是 n 很大時的成長趨勢，並不表示每一個 n 都一定比較快。

若甲演算法雖為 O(n log n)，但常數成本很大，例如需要遞迴、額外記憶體配置、複雜資料結構或較差的快取區域性；而乙演算法雖為 O(n^2)，但實作簡單、常數小、資料都在連續陣列中，則在小型測試資料上乙的實際執行時間可能反而較短。

二

Complete Binary Tree 節點數界線證明

配分 20 分

題目

何謂 Complete Binary Tree？假設單一節點的樹其樹高為 1，證明若 Complete Binary Tree 含 n 個節點且樹高為 h，則 2^(h-1) ≤ n ≤ 2^h - 1。

定義

Complete Binary Tree（完全二元樹）是指除了最後一層外，每一層都必須填滿；最後一層的節點則由左至右依序填入，不可中間跳空。

下界證明

若樹高為 h，代表第 h 層至少有一個節點。因為完全二元樹在最後一層以前都必須填滿，所以第 1 層到第 h - 1 層共有 1 + 2 + 4 + ... + 2^(h-2) = 2^(h-1) - 1 個節點。

再加上第 h 層至少 1 個節點，可得 n ≥ 2^(h-1)。

上界證明

高度為 h 的二元樹若每一層都填滿，節點數最多。此時總節點數為 1 + 2 + 4 + ... + 2^(h-1) = 2^h - 1。

因此對任一高度為 h 的 Complete Binary Tree，都有 2^(h-1) ≤ n ≤ 2^h - 1。

若用 multilist 計算所有節點 degree，可沿著每個頂點的邊串列計數，整體時間為 O(n + e)；儲存頂點表與邊節點所需空間也是 O(n + e)。

比較上，adjacency matrix 查詢兩點是否相鄰很快，且適合稠密圖，但空間固定為 n^2。Adjacency multilist 較適合稀疏圖，空間隨實際邊數成長，但查詢任兩點是否相鄰通常需要沿邊串列搜尋。

C++ 變數作用域、傳值與傳參考

配分 30 分

題目

請追蹤下列 C++ 程式，列出輸出結果，並說明每項輸出對應數值的原因。 

#include <iostream>
int a = 10;
void testA(int a) {
  cout << "1. a = " << a << endl;
  a++;
  cout << "2. a = " << a << endl;
}
void testB(void) {
  cout << "3. a = " << a << endl;
  a++;
  cout << "4. a = " << a << endl;
}
void testC(int &a) {
  cout << "5. a = " << a << endl;
  a++;
  cout << "6. a = " << a << endl;
}
int main() {
  int a = 100;
  cout << "7. a = " << a << endl;
  a++;
  testC(a);
  cout << "8. a = " << a << endl;
  a++;
  testB();
  cout << "9. a = " << a << endl;
  a++;
  testA(a);
  cout << "10. a = " << a << endl;
  return 0;
}

輸出結果

7. a = 100
5. a = 101
6. a = 102
8. a = 102
3. a = 10
4. a = 11
9. a = 103
1. a = 104
2. a = 105
10. a = 104

原因說明

全域變數 a 初值為 10；main 內又宣告區域變數 a = 100，在 main 中會優先使用此區域變數。

main 先輸出 100，接著 a++ 變成 101。呼叫 testC(a) 時使用參考傳遞，因此函式內的 a++ 會直接改到 main 的區域變數，所以依序輸出 101、102，回到 main 後第 8 行仍為 102。

接著 main 的 a++ 變成 103。呼叫 testB() 時，函式內沒有區域參數 a，所以使用全域變數，輸出 10 後把全域變數改成 11，再輸出 11；這不影響 main 的區域變數。之後第 9 行輸出 103，再遞增為 104。testA(a) 是傳值呼叫，參數得到副本 104，函式內輸出 104、105，但不會改變 main 的 a，所以最後第 10 行輸出 104。

兩矩形是否重疊的判斷邏輯

配分 20 分

題目

給定方形 A 的左上角座標 (x1, y1)、寬高 (w1, h1)，以及方形 B 的左上角座標 (x2, y2)、寬高 (w2, h2)，請設計判斷邏輯確認兩方形是否有重疊，有重疊回傳 true，否則回傳 false。

判斷想法

以常見螢幕座標為例，左上角為起點，x 往右增加，y 往下增加。兩矩形不重疊只有四種情況：A 完全在 B 左邊、A 完全在 B 右邊、A 完全在 B 上方、A 完全在 B 下方。只要不是這四種情況，就代表有面積重疊。

虛擬碼

boolean overlap(x1, y1, w1, h1, x2, y2, w2, h2):
  if x1 + w1 <= x2:
    return false
  if x2 + w2 <= x1:
    return false
  if y1 + h1 <= y2:
    return false
  if y2 + h2 <= y1:
    return false
  return true

這裡使用 <= 表示若兩矩形只是邊界相接，沒有共同面積，則視為不重疊。若題目把「碰到邊」也算重疊，可將 <= 改為 <。

Pascal 遞迴呼叫、記憶化與分支圖

配分 25 分

題目

以圖示法表示遞迴程式呼叫執行過程，給定 n = 4、k = 2 呼叫 pascal(4,2)，求最後回傳結果，並依產生的堆疊與分支圖呈現呼叫過程。 

R = {}
def pascal(n, k):
  if k == 0 or k == n:
    return 1
  if (n, k) in R:
    return R[(n, k)]
  R[(n, k)] = pascal(n-1, k-1) + pascal(n-1, k)
  return R[(n, k)]

分支圖與回傳值

pascal(4,2)
├─ pascal(3,1)
│  ├─ pascal(2,0) = 1
│  └─ pascal(2,1)
│     ├─ pascal(1,0) = 1
│     └─ pascal(1,1) = 1
│     ⇒ R[(2,1)] = 2
│  ⇒ R[(3,1)] = 1 + 2 = 3
└─ pascal(3,2)
   ├─ pascal(2,1) = 2  (由 R 直接取出)
   └─ pascal(2,2) = 1
   ⇒ R[(3,2)] = 2 + 1 = 3

R[(4,2)] = 3 + 3 = 6

因此最後回傳值為 6。其中 pascal(2,1) 第一次計算後被存入 R，第二次遇到相同參數時直接取出，可避免重複展開遞迴。

以隱私保護方式比較財富

配分 25 分

題目

張三和李四要比較誰較有錢，但不直接說出財富金額。假設一個陣列序號代表一個財富單位，擁有該單位則標記為 T。張三擁有 4 千萬，標記 1 到 4 為 T；李四擁有 6 千萬，標記 1 到 6 為 T。請設計虛擬碼，讓兩人運用此程式驗證誰較有錢。

1234567
張三TTTT---
李四TTTTTT-

比較原理

若 A 有 a 個單位，A 只需查看 B 的第 a 個格子：若是 T，表示 B 至少有 a 個單位；若不是 T，表示 B 少於 a 個單位。同理，B 只需查看 A 的第 b 個格子。雙方不需要公開完整金額，只需要公開必要的檢查結果。

虛擬碼

function comparePrivate(A_name, A_array, a, B_name, B_array, b):
  A_sees_B = B_array[a]
  B_sees_A = A_array[b]

  if A_sees_B == 'T' and B_sees_A != 'T':
    return B_name + " 較有錢"

  if A_sees_B != 'T' and B_sees_A == 'T':
    return A_name + " 較有錢"

  if A_sees_B == 'T' and B_sees_A == 'T':
    return "兩者財富相同"

  return "輸入資料不一致，需重新檢查"

代入題目資料時，張三查看李四第 4 格，結果為 T，表示李四至少有 4 千萬；李四查看張三第 6 格，結果為 -，表示張三不到 6 千萬。因此可判定 李四較有錢。
